# RunningHub 批量任务助手

在 RunningHub 工作流页面加一个悬浮面板，**自动识别两类工作流**：

- **视频换装模式**（工作流里有 `VHS_LoadVideo` 节点）：图片+视频按视频为主 1 配 1，有限批量。
- **纯图片模式**（没有视频节点，如 Z-Image-Turbo）：只上传图片→运行→出图，可**无限循环**（顺序/随机），自动下载成品图。

两种模式都支持：选目录（刷新后可续跑）、从第几个开始、运行模式、**卡住 N 分钟自动刷新续跑**、
回收成品（`SaveImage` 图片 / `VHS_VideoCombine` 视频，成功失败都下载）。

---

## 视频换装模式

选一次本地文件夹，扩展就按顺序把图片+视频**成对**处理，每组自动完成：

1. 调 ComfyUI 的 `/upload/image` 接口上传图片、视频到输入目录
2. 把返回的文件名写进工作流里的 **`LoadImage` 节点（图片）** 和 **`VHS_LoadVideo` 节点（视频）**
3. 点顶层「运行」按钮创建任务
4. 自动下一组

> 为什么不能直接“填本地路径”？浏览器出于安全限制，不允许网页读取任意磁盘路径，
> 只能让你手动选一次文件夹。选完之后全自动。**不需要 API Key**，复用你已登录的会话。

> 工作原理：这个页面是完整的 ComfyUI 编辑器，嵌在同源 iframe 里。扩展以 `MAIN` 世界注入，
> 进 iframe 拿到 ComfyUI 的 `app` 对象，直接走节点级 API，比模拟点击稳得多。
> 已按你的工作流校准：图片节点 `id 332 (LoadImage)`、视频节点 `id 52 (VHS_LoadVideo)`。

## 一、安装

独立 Chrome / Edge 扩展。文件夹：`runninghub_extension/`（含 `manifest.json`、`content.js`、`icon.png`）

1. 打开浏览器扩展页：
   - Chrome：地址栏输入 `chrome://extensions`
   - Edge：地址栏输入 `edge://extensions`
2. 右上角打开 **开发者模式 / Developer mode**。
3. 点 **加载已解压的扩展 / Load unpacked**，选中 `runninghub_extension` 这个文件夹。
4. 打开工作流页面 `https://www.runninghub.ai/workflow/2061100845663608833`（`.cn` 也行），
   右上角会出现深色的「RH 批量任务」面板。
   - 改了 `content.js` 后，回扩展页点该扩展的 **刷新/重新加载** 图标，再刷新网页即可生效。

> 配置项都在 `runninghub_extension/content.js` 顶部的 `CONFIG`。

## 二、准备素材文件夹

准备**两个独立文件夹**：一个放图片，一个放视频。

```
图片文件夹/                 视频文件夹/
├── 001.jpg               ├── 001.mp4
├── 002.jpg               ├── 002.mp4
└── ...                   └── ...
```

- **以视频为主序列**：视频 1000 个、图片 30 个也行。每个视频按你选的方式取一张图片配对：
  - **顺序循环**：第 v 个视频 → 第 `v % 图片数` 张图。
  - **随机**：固定种子随机取（刷新续跑后配对不变）。
- 两个文件夹各自**按文件名自然排序**（`001 < 002 < ... < 010`）。
- 可指定**从第几个视频开始**（补跑用）。
- 图片节点 = `332 (LoadImage)`，视频节点 = `52 (VHS_LoadVideo)`。

> 文件夹用的是“目录授权”，所以**刷新页面后还能自动恢复并续跑**。

## 三、使用

1. 打开工作流页面，**等画布加载完**——面板顶部显示「ComfyUI：已连接 ✓」后才可开始。
2. 分别点 **📁 图片文件夹** 和 **🎬 视频文件夹**（会弹目录授权，点“允许/查看”）。
3. 设置：**从第几个视频开始**、**图片配对**（顺序循环/随机）、**运行模式**（见下）。
4. 点 **📥 选择保存目录** 选保存文件夹；下方勾选要回收成品的视频节点（默认已勾 226）。
5. 点 **开始**。每组自动：上传图→写 332 → 上传视频→写 52 → 点运行 → 等成品→**下载到保存目录**（服务器原名）→ 下一组。
6. 过程中可 **暂停 / 继续 / 停止**。某条失败会自动暂停，处理后点「继续」重试当前条。

### 运行模式（Lite/Plus）

- **推荐做法**：开始前在页面上**手动**把模式选成 `Lite/Plus`，会一直生效，零风险。
- 想自动切：面板“运行模式”框填 `Lite/Plus`。但你页面有两个都含“运行”的按钮，自动点**有重复运行风险**，
  请先放 1 条试确认不会跑两次。默认该框为空（不自动切）。

### 超时自动刷新 + 断点续跑

- ComfyUI **超过 20 分钟无任何进度**（真卡死，不是正常生成）→ 扩展**保存断点并自动刷新**，刷新后从中断处继续。
- 刷新后若需重新授权目录，会出现绿色 **「▶ 继续上次（从第 N 个）」** 按钮，点一下即可；权限仍在则全自动续跑。
- 阈值在面板「卡住刷新 N 分钟」里改（视频换装建议 20，纯图片建议 2）。

## 纯图片无限模式（Z-Image-Turbo 等没有视频节点的工作流）

1. 等「ComfyUI：已连接 ✓」。扩展检测到没有视频节点，自动进入**纯图片模式**（视频文件夹忽略）。
2. 点 **📁 图片文件夹** 选输入图片所在目录（会列出图片张数）。
3. 设置：
   - **从第几个开始**：默认 1。
   - **取图方式**：`顺序循环`（按文件名顺序，跑完一遍再从头）或 `随机`。
   - 勾上 **无限循环**：一直跑不停（不勾＝把图片跑一遍就停）。
   - **卡住刷新**：建议填 `2` 分钟（出图快，卡住就刷新续跑）。
4. 点 **📥 选择保存目录**；下方勾选输出节点（`SaveImage` 默认已勾，会下载成品图，服务器原名）。
5. 点 **开始**。每轮：上传一张图→写 `LoadImage`→运行→等成品图→下载→下一张……无限循环。
6. 卡住超时会自动刷新并从断点续跑；想停就点 **停止**。

> 图片/视频输入节点都是**按类型自动识别**的（`LoadImage` / `VHS_LoadVideo`），换工作流不用改 id。
> 这个工作流末尾也有 `PlaySound` 会让任务报“失败”，但 `SaveImage` 早就出图了，照样能下载到。

> **关于“回收成品视频”**：你这个工作流末尾的节点（如 `PlaySound`）会让任务报“失败”，
> 但 `VHS_VideoCombine` 早就把视频生成好了。扩展不管成功/失败，只要被勾选节点出现**新的成品文件名**
> 就把视频下载下来。下载优先走同源 `/view` 接口（带你的登录态），失败再退回 `cos_url` 直链。
> 选了保存目录就写进该目录，没选则进浏览器默认下载文件夹。

## 四、调参（`content.js` 顶部 `CONFIG`）

| 配置项 | 作用 |
|---|---|
| `imageNodeId` / `videoNodeId` | 默认 `null`＝按类型自动识别（适配多个工作流）；也可填 id 精确匹配 |
| `imageNodeType` / `videoNodeType` | 自动识别用的节点类型（`LoadImage` / `VHS_LoadVideo`） |
| `imageWidget` / `videoWidget` | 节点上承载文件名的 widget 名（`image` / `video`） |
| `outputNodeTypes` | 可回收成品的节点类型（`SaveImage`/`VHS_VideoCombine`/预览…） |
| `runMode` | 自动选运行模式（默认 `""` 不切）。填 `Lite/Plus` 等可尝试自动切 |
| `runButtonText` / `runButtonClass` | 运行按钮的文字/类名匹配 |
| `confirmButtonText` | 消耗确认弹窗按钮文字 |
| `uploadTimeoutMs` | 单个文件上传超时（视频大就调大） |
| `outputWaitMs` | 等成品视频出现的最长时间（默认 20 分钟） |
| `afterRunDelayMs` | 点运行后先等多久 |
| `waitButtonReenableMs` | 等运行按钮重新可用的最长时间（串行排队） |
| `perTaskGapMs` | 两个任务之间额外间隔 |

> 面板上的“从第几个开始 / 取图方式 / 无限循环 / 卡住刷新分钟 / 运行模式”都是运行时选项，不用改 `CONFIG`。
> 节点 id 默认就是 `null` 自动识别，换工作流一般不用动。

## 五、排错

- **「开始」按钮是灰色点不了**：把鼠标移到按钮上会显示原因。常见：
  - “等待 ComfyUI 画布加载完”——等顶部变「已连接 ✓」。
  - “视频换装模式：需要同时选好图片和视频文件夹”——两个都要选。
  - “请先选图片文件夹”（纯图片模式）。
  - “正在运行中”——上一个批量还在跑（比如刚从另一个工作流切过来）。点“停止”或等它结束。
- **两个工作流互相影响 / 切换工作流后状态乱**：扩展按**工作流 id 分开保存**设置和断点，且检测到 SPA 切换会自动停掉上一个批量、重置面板。若仍异常，**整页刷新**（F5）一下最干净。
- **模式认错了**：面板「工作流模式」默认“自动检测”。可手动改成「视频换装」或「纯图片」。
- **面板没出现**：确认扩展已启用、网址匹配 `runninghub.ai`/`.cn`，刷新页面。
- **一直显示「ComfyUI：等待画布加载…」**：画布还没初始化完，或 `manifest.json` 里没设 `"world": "MAIN"`（必须有，否则拿不到 iframe 的 `app`）。
- **“没找到图片/视频节点”**：工作流被改过、节点 id 变了。把 `imageNodeId`/`videoNodeId` 设为 `null` 自动识别，或重新跑 `导出节点_控制台片段.js` 把新 id 发我。
- **“上传失败 HTTP 4xx/5xx”**：多半是会话过期或余额问题；先在页面手动上传一次确认账号正常。
- **点了运行没反应 / 点错按钮**：调 `runButtonText`、`runButtonClass`。这个工作流主运行按钮文字是「运行」、类名含 `run-btn`。
- **每次只跑一条就卡很久**：Wan 视频生成很慢且可能限并发。扩展会等运行按钮恢复后再提交下一条；嫌慢可减小 `waitButtonReenableMs`，但可能会撞上并发上限。

## 六、把真实页面结构导出给作者校准（重要）

我看不到你登录后的真实页面，选择器是按通用规则猜的。如果不灵，请按下面任一方式把
**真实 DOM** 发我，我精调后基本就能一把过。

### 方式 1（推荐，体积小、最有用）：控制台导出控件信息

1. 登录后打开工作流页面，**让上传图片/视频的控件和“运行”按钮都显示出来**。
2. 按 `F12` 打开开发者工具 → 切到 **Console（控制台）** 标签。
3. 把项目里 `导出DOM_控制台片段.js` 的内容整段粘进去，回车。
4. 控制台会打印一段 JSON 且自动复制到剪贴板；在 Cursor 里新建文件 `live_dom.json` 粘贴保存，发我。
   - 它会列出：所有 `<input type=file>`（含 accept）、所有候选按钮文字、是否有 iframe（ComfyUI 常嵌在 iframe 里）。

### 方式 2：保存整页渲染后的 HTML

1. 同样先登录并让控件显示出来。
2. `F12` → Console，运行：`copy(document.documentElement.outerHTML)`
3. 新建 `live_dom.html` 粘贴保存，发我。
   - 注意：若控件在 iframe 里，这种方式抓不到 iframe 内部，用方式 1 的结果会更准。

### 方式 3：截图

直接把“上传图片/视频 + 运行按钮”那块区域截图发我，配合方式 1 一起最好。
